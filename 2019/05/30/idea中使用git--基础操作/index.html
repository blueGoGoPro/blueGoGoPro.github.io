简介

idea对git做了很好的支持。在idea中，我们可以完成全部的常见git版本管理操作，无需命令行或者其他的git客户端。本文将介绍版本管理各环节中需要的操作及按钮位置等。纯新手向文章，适合第一次接触idea+git的同学了解入门。

检出

第一步：从远程仓库获取代码，步骤如下

填写 git仓库地址,选择导入的本地文件夹,填写文件夹名称 ,点击clone,即可将远程的仓库的内容clone到本地.

提交

有内容提交时：点击右上角的绿色按钮（对勾或者向上的箭头，不同idea版本按钮不同）

里面的提交内容的勾选,以及填写提交信息的内容一样.但下方commit按钮和svn含义不同,

Git 中的commit 是提交到本地的版本库中,并不会同步到远程仓库中.如果要提交到远程仓库,需要点击commit后面的小三角,选择commit and push,在接下来的弹框中点击push,即可将提交推送至远程仓库.

更新

同样:点击idea右上角的vcs向下的箭头,弹出的对话框中点击ok即可.

默认选中的Merge 和using Stash 分别代表合并到当前分支和清空工作区.适用.

解决冲突

1.更新时冲突

如果当前代码和远程库的代码中存在冲突,idea会弹框警告,操作流程和svn一样,视情况选择右边的三个选项:

Accept Yours: 完全使用你的代码，这将覆盖远程分支的代码，

Accept Theirs : 完全使用远程分支的代码，这将覆盖你的代码

Merge ：手工合并

如果不是特别清晰本地代码和远程代码冲突的原因以及解决方案，建议使用Merge方式手工合并。

点击Merge，会弹出如下画面。

由左至右三列代码分别是： 你的代码，合并后结果代码，远程仓库的代码。

idea会标记出不一样的地方。’>>’ 代表将当前代码应用到结果集，’X’ 代表舍弃当前代码。酌情取舍即可

2.提交时冲突

如果提交时本地代码和远程仓库有冲突,idea会提示,并要求你推送之前解决冲突

此时:点击merge,idea会先执行更新操作(pull),将远程仓库代码拉取到本地,然后弹出和更新时冲突一样的弹框,点击操作就好.

如果解决冲突失败,再次点击提交时发现:

这是因为代码已经提交至本地仓库,所以会显示没有可以提交的内容,但实际上内容并未推送到远程仓库,需要手动推送

在需要提交的地方右键,如图按顺序选择即可.

分支

以上操作都是基于当前分支操作的,且当前分支已经关联到远程仓库对应的分支.

idea可以在导航栏cvs处和右下角git处操作git分支：包含远程分支和本地分支。在对应的分支下也有相应的checkout 和merge操作

其中:new branch 是创建新的分支

Local branches 是指本地的其他分支,最下面一行小字, current branch: 后面才是本地的当前分支,remoteBranches 是指关联的远程库上的分支.

切换本地分支:点击对应本地分支后面的小三角,选择checkout即可切换到对应分支,此时在idea中修改,就是在对应的分支下修改.
